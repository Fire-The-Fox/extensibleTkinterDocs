Extensible Tkinter

What actually is Xtk?

Xtk is markup language created on top of XML with ability to construct GUI thanks to Builder class from tkinter-expansion.

With this concept in mind, you can even create builder application that can generate Xtk for you, with ease rather than just generating the code.

Not only tkinter-expansion but also many other popular GUI frameworks support and preffer xml for GUI construction

List of popular GUI frameworks that use xml:

  • Android app development ( java/kotlin )

  • JavaFX ( java/kotlin/groovy )

  • UWP and WPF ( .NET C#, windows only )

  • Avalonia ( .NET C# )

  • Gtk ( C )

  • QT ( C++ )

List of all supported tags by Xtk

Button # Widget which allows user input from mouse and can trigger function
Canvas # Widget on which you can draw shapes
CheckButton # Square based widget used in most of every forums
Entry # Widget into which you can write but only one line is supported
Frame # Widget capable of containing other widgets
Label # Widget for displaying text onto the screen
LabelFrame # Combination of Label and Frame widget, in short: You have Frame and title on top of it
Listbox # Sort text in vertical order
Menu # Menu bar on top of application
PannedWindow # Split widgets in one widget
Radiobutton # Circle based widget used in most of every forums
Scale # Widget which allows you to select value from range
Scrollbar # Classic scrollbar widget which can control specific view coordinate of widget
Text # Widget on which you can write text
Tk # Can be used only once, this widget must be root of Xtk file
TopLevel # It's similar to Tk, but you can have them more than one

Xtk tag rules

Every single tag is name of widget, for example: Button
Every single tag must have a name attribute otherwise Builder class is going to complain
Other tag than Tk is not allowed, otherwise it might cause troubles
Tag should contain parent attribute
Only one specific placement method is allowed per widget: place, grid or pack

Xtk example

main.xtk 
<?xml version="1.0" encoding="utf-8" ?>
<Tk name="root" width="1080" height="1080" title="Weather App" resizable="False False">
    <Label name="Background" parent="root" bg="#9DC7F9">
        <place relwidth="1" relheight="1"> </place>
    </Label>

    <Frame name="WBackground" bg="white">
        <place x="180" y="180" width="720" height="720"> </place>
    </Frame>

    <Label name="WText" parent="WBackground" text="Type weather location: " bg="white" font="Calibri" fontsize="25">
        <place x="25" y="25"> </place>
    </Label>

    <Entry name="WEntry" parent="WBackground" bg="white" highlightbackground="#9DC7F9" highlightthickness="10" highlightcolor="#9DC7F9" font="Calibri" fontsize="30">
        <place x="25" y="90" height="90" width="495"> </place>
    </Entry>

    <Button name="WSearch" parent="WBackground" text="Search" bg="white" highlightcolor="white" highlightthickness="10" relief="sunken" bd="0" highlightbackground="#9DC7F9" font="Calibri" fontsize="25">
        <place x="520" y="90" height="90" width="180"> </place>
    </Button>

    <Label name="WData" parent="WBackground" bg="white" highlightbackground="#9DC7F9" highlightthickness="10" justify="left" anchor="nw" font="Calibri" fontsize="23">
        <place x="25" y="205" height="490" width="675"> </place>
    </Label>
</Tk>
main.py 
import extensibleTkinter as tke


builder = tke.Builder(file="gui.xtk")
builder.init()
builder.generate()

WEntry = builder.variables["WEntry"] # As you can see, I am accessing variables by their names which are generated by the Builder
WSearch = builder.variables["WSearch"]
WData = builder.variables["WData"]

builder.variables["root"].mainloop()
